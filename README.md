# 📊 Mobile Sales Performance Dashboard – Power BI

## 📌 Project Overview
This project analyzes mobile sales performance across multiple cities, brands, payment methods, and customer ratings using Power BI.  
The objective was to identify revenue drivers, customer behavior patterns, and performance trends.

---

## 📂 Dataset Details
- Total Sales: 769M
- Total Quantity Sold: 19K
- Total Transactions: 4K
- Average Quantity per Transaction: 4.99
- Data fields include: City, Brand, Model, Payment Method, Ratings, Month, Sales, Quantity

---

## 📊 Dashboard Features
- KPI Cards (Total Sales, Total Quantity, Transactions, Average Quantity)
- Month-wise slicer filtering (January–December)
- City-level sales map visualization
- Brand-wise contribution analysis
- Payment method distribution (UPI, Debit Card, Credit Card, Cash)
- Day-wise quantity trend analysis
- Customer rating distribution funnel
- Product-level performance comparison

---

## 📈 Key Insights
- UPI contributed the highest share of transactions (~26%), indicating strong digital payment adoption.
- Apple were among the top revenue-generating brands.
- Weekend sales showed higher volume compared to mid-week performance.
- Certain cities significantly outperformed others in total sales contribution.
- Average quantity per transaction remained stable around 5 units.
